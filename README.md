# NetGuard Pro
> **تحكم كامل بشبكتك المنزلية بثقة وأمان**

---

## 📝 وصف المشروع
**NetGuard Pro** هو تطبيق احترافي يعمل على جهازك المحلي (localhost)، مصمم ليوفر لك رقابة كاملة وشاملة على شبكتك المنزلية. يقوم التطبيق باكتشاف نوع الراوتر المتصل تلقائياً، ويعرض قائمة حية بجميع الأجهزة المتصلة مع قياس دقيق لاستهلاك كل جهاز للإنترنت (Bandwidth).

سواء كنت مبتدئاً أو تقنياً، يمنحك التطبيق الأدوات اللازمة لحظر الأجهزة غير المصرح بها، إعادة تشغيل الراوتر عن بُعد، ومراقبة نشاط الشبكة لحظة بلحظة لضمان أقصى درجات الأمان والخصوصية.

### ميزات التطبيق الرئيسية:
*   **اكتشاف تلقائي للراوتر:** لا حاجة لإعدادات يدوية معقدة؛ التطبيق يتعرف على بوابة الشبكة فور تشغيله.
*   **تحليل دقيق للاستهلاك:** عرض بياني مفصل (شامل وجزئي) لاستهلاك البيانات لكل جهاز.
*   **دعم متعدد اللغات:** واجهة كاملة باللغتين العربية (RTL) والإنجليزية (LTR).
*   **متعدد المنصات:** يمكن استخدامه عبر المتصفح، أو تثبيته كتطبيق سطح مكتب (Windows) أو تطبيق هاتف (Android).
*   **سجل أمان مدمج:** تسجيل كامل لعمليات البناء والنشاط لضمان الشفافية.

---

## ✨ الميزات الأساسية
- **اكتشاف تلقائي:** فحص ذكي للشبكة للتعرف على موديل الراوتر ونوعه.
- **دعم واسع للرواترات:** يدعم TP-Link, Huawei, D-Link, ASUS, Linksys, Keenetic وغيرها.
- **مراقبة حية (Real-time):** تحديث فوري لبيانات الأجهزة عبر تقنية WebSockets.
- **أمان فائق:** نظام دخول محمي بـ JWT وتشفير محلي للبيانات الحساسة باستخدام AES-256.
- **واجهة مستخدم متطورة:** دعم كامل للوضع الليلي والنهاري مع تصميم عصري (Glassmorphism).
- **CI/CD احترافي:** بناء تلقائي عبر GitHub Actions لإنتاج ملفات EXE و APK مع كل تحديث.
- **سجل تشخيص مدمج:** عرض سجلات العمليات (Process Logs) مباشرة من داخل الواجهة.

---

## 🛠 التقنيات المستخدمة
- **Backend:** Node.js + Express + Socket.io + Python (Scrapy, requests, paramiko)
- **قاعدة بيانات:** SQLite (محلية ومشفرة)
- **Frontend:** React + TypeScript + Tailwind CSS + i18next
- **سطح المكتب:** Electron
- **الهاتف:** Capacitor
- **الأمان:** Helmet, express-rate-limit, JWT, Dependabot
- **البناء:** GitHub Actions

## 🚀 خطوات التثبيت والتشغيل

اتبع الأوامر التالية بالترتيب في نافذة Terminal:

1. **استنساخ المستودع:**
   ```bash
   git clone https://github.com/username/NetGuard-Pro.git
   ```

2. **الدخول للمجلد:**
   ```bash
   cd NetGuard-Pro
   ```

3. **تثبيت اعتماديات الخادم الخلفي (Backend):**
   ```bash
   cd backend/node_server && npm install
   ```

4. **تثبيت اعتماديات بايثون (Python Core):**
   ```bash
   cd ../python_core && pip install -r requirements.txt
   ```

5. **تثبيت اعتماديات الواجهة (Frontend):**
   ```bash
   cd ../../frontend && npm install
   ```

6. **إعداد ملف البيئة (.env):**
   أنشئ ملفاً باسم `.env` في مجلد `backend/node_server` وضع فيه:
   ```env
   JWT_SECRET=secret123
   PORT=3001
   DB_PATH=../data/netguard.db
   ```

7. **تشغيل الخادم الخلفي:**
   (من داخل مجلد `backend/node_server`)
   ```bash
   npm run dev
   ```

8. **تشغيل واجهة المستخدم:**
   (من داخل مجلد `frontend`)
   ```bash
   npm run dev
   ```

9. **الوصول للتطبيق:**
   افتح المتصفح على: `http://localhost:5173`
   *بيانات الدخول الافتراضية:* المستخدِم: `admin` / كلمة المرور: `admin` (يرجى تغييرها فوراً).

## 📦 كيفية بناء التطبيق للنشر

- **لبناء تطبيق Windows (EXE):**
  ```bash
  cd desktop && npm run build
  ```
  ستجد الملف الناتج في مجلد `desktop/dist/`.

- **لبناء تطبيق Android (APK):**
  (يتطلب وجود Android SDK و Capacitor)
  ```bash
  cd mobile && npm run build
  ```

- **عبر GitHub Actions:**
  بمجرد دفع الكود (Push) إلى فرع `main` المنشور على GitHub، سيقوم النظام تلقائياً ببناء الإصدارات. يمكنك تحميلها من قسم `Actions` -> اختر آخر عملية -> `Artifacts`.

## 🔒 الأمان والخصوصية
*   **خصوصية مطلقة:** التطبيق يعمل محلياً 100%؛ بياناتك لا تغادر جهازك أبداً ولا يتم إرسالها لأي خوادم خارجية.
*   **تشفير قوي:** يتم تشفير كلمات المرور ومفاتيح الدخول محلياً قبل التخزين.
*   **تحديثات دورية:** نستخدم **Dependabot** لتحديث التبعيات ومنع الثغرات الأمنية بشكل أسبوعي.
*   **حماية من الثغرات:** تم إصلاح ثغرات Vite (CVE-2026-39365) وتأمين المكتبات ضد تنفيذ الأكواد عن بُعد (RCE).
*   **تنبيه:** أمان شبكتك يبدأ منك؛ غير كلمة المرور الافتراضية `admin` بمجرد تسجيل دخولك الأول.

## 📂 هيكل المشروع
*   `backend/`: يحتوي على خادم Node.js وموديلات التحكم المكتوبة ببايثون.
*   `frontend/`: يحتوي على واجهة المستخدم المبنية بـ React و TypeScript.
*   `desktop/`: إعدادات Electron لتحويل التطبيق لنسخة سطح مكتب.
*   `mobile/`: إعدادات Capacitor لتحويل التطبيق لنسخة أندرويد.
*   `scripts/`: نصوص برمجية مخصصة لعمليات البناء الفني والأمان.
*   `build_logs/`: المجلد الذي يتم فيه تخزين سجلات النشاط `system.log`.

## 📚 قسم المبتدئين
- **ما معنى localhost؟** هو عنوان وهمي يشير إلى جهازك الشخصي فقط، لا يمكن لأي شخص خارج شبكتك الوصول إليه.
- **ما الفرق بين API و WebSocket؟** الـ API يشبه طلب الوجبة (ترسل طلباً وتنتظر رداً)، أما WebSocket فهو خط مفتوح دائم يسمح للسيرفر بإرسال البيانات لك فور حدوثها (مثل تحديث السرعة والتحميل).
- **ما هو JWT؟** هو رمز رقمي مشفر يثبت هويتك للتطبيق دون الحاجة لإرسال كلمة المرور مع كل طلب.
- **كيف تعمل GitHub Actions؟** هي روبوتات برمجية تقوم بتحويل الكود البرمجي إلى تطبيق جاهز والتشغيل (EXE/APK) تلقائياً بمجرد إرسال الكود للمستودع.
