# NetGuard Pro V7 🛡️

**NetGuard Pro** هو تطبيق Native متطور وشامل لإدارة الشبكات وأمن أجهزة التوجيه (Routers)، مبني بالكامل باستخدام لغة **Python** وإطار عمل **Flet**. تم تصميمه ليوفر واجهة عصرية (Material 3) للتحكم الكامل في بيئة الشبكة، مع دمج ذكاء اصطناعي متقدم (Gemini) لتعزيز الأمان.

---

## 📁 هيكل المشروع بالتفصيل (Detailed Project Structure)

| الملف / المجلد | الوصف الوظيفي الدقيق |
| :--- | :--- |
| `main.py` | **المحرك الرئيسي (Backend & UI)**: الكود المصدري بلغة بايثون الذي يدير واجهة المستخدم الرسومية (Flet)، عمليات الاتصال ببروتوكولات HTTP/HTTPS، محرك الفحص الأمني للمنافذ، ونظام تحليل البيانات عبر Gemini AI. |
| `src/App.tsx` | **نظام العرض المتطور (Web Preview)**: كود React/TypeScript الذي يحاكي تجربة التطبيق الحقيقي بدقة 100% داخل المتصفح، بما في ذلك الرسوم البيانية التفاعلية (Recharts) ونظام التبديل اللغوي. |
| `netguard_pro.db` | **مخزن البيانات (SQLite)**: قاعدة بيانات محلية تخزن تاريخ استهلاك البيانات، سجلات التهديدات الأمنية، وإعدادات التطبيق بشكل دائم. |
| `secret.key` | **مفتاح الأمان**: ملف يحتوي على مفتاح تشفير عشوائي (Fernet) يُستخدم لتأمين بيانات الراوتر الحساسة داخل قاعدة البيانات لضمان عدم وصول أي طرف خارجي إليها. |
| `requirements.txt` | **قائمة المتطلبات**: يحدد الإصدارات الدقيقة للمكتبات البرمجية (مثل `flet`, `httpx`, `cryptography`) لضمان عمل التطبيق في أي بيئة تطوير. |
| `.github/workflows/build.yml` | **مصنع التطبيقات (CI/CD)**: ملف أتمتة يقوم ببناء التطبيق وتحويله إلى ملفات تنفيذية (APK للأندرويد و EXE للويندوز) عند كل تحديث للكود. |
| `metadata.json` | **بيانات التعريف**: يحدد اسم التطبيق، وصفه، والأذونات التقنية التي يحتاجها (مثل الوصول للشبكة) ليعمل بشكل صحيح في بيئة السحابة. |
| `tailwind.config.js` | **تنسيق الواجهة**: إعدادات إطار عمل Tailwind CSS المستخدم لتصميم واجهة المعاينة الويب بشكل عصري وتقني. |
| `vite.config.ts` | **محرك المعاينة**: إعدادات أداة Vite التي تضمن سرعة تشغيل وتحديث نظام المعاينة الويب أثناء التطوير. |
| `package.json` | **إدارة حزم الويب**: يحتوي على تعريفات المكتبات المستخدمة في نظام المعاينة مثل `Lucide React` للأيقونات و `Recharts` للرسوم البيانية. |
| `.env.example` | **دليل الإعداد**: ملف توضيحي يخبر المطور بالمتغيرات المطلوبة لتفعيل ميزات الذكاء الاصطناعي (مثل مفتاح Gemini). |
| `.gitignore` | **قائمة الاستبعاد**: يمنع رفع الملفات المؤقتة أو الحساسة (مثل مفاتيح التشفير الشخصية) إلى مستودع الكود العام. |

## 🏗️ البنية التقنية (Technical Stack)

- **Core Logic**: Python 3.11.
- **UI Framework**: [Flet](https://flet.dev) (يعتمد على Flutter لتقديم أداء Native).
- **Security Engine**: فحص منافذ (Port Scanner) واكتشاف هجمات حقيقي.
- **Database**: SQLite مشفر محلياً.
- **AI Integration**: Google Gemini API للتحليل الأمني الذكي.
- **CI/CD**: GitHub Actions للبناء التلقائي لملفات APK و EXE.

---

## 🚀 الميزات الرئيسية

### 1. لوحة تحكم تقنية (Technical Dashboard)
- **تصميم احترافي**: واجهة مستوحاة من غرف التحكم العالمية مع دعم كامل للغة العربية (RTL).
- **مراقبة الحالة**: عرض لحظي لحالة النظام، وقت التشغيل (Uptime)، والتأخير (Latency).
- **إحصائيات الاستهلاك**: تتبع دقيق لحجم البيانات المستهلكة مع رسوم بيانية توضيحية.

### 2. الدرع الأمني (Security Shield)
- **فحص المنافذ**: التأكد من إغلاق المنافذ الخطيرة في الراوتر (مثل Telnet).
- **اكتشاف MITM**: التنبيه الفوري في حال تغير عنوان الـ Gateway بشكل مريب.
- **سجل الأمان**: تسجيل كافة الأحداث الأمنية في قاعدة بيانات مشفرة.

### 3. المساعد الذكي (AI Assistant)
- **نصائح مخصصة**: يقوم Gemini بتحليل سجلات الأمان والاستهلاك لتقديم نصائح احترافية بالعربية.
- **دعم الحملات**: يركز الذكاء الاصطناعي على دعم حملة "إنترنت غير محدود" عبر تحسين الاستهلاك.
