# 📑 تقرير تحليل مشروع luci-mobile-1.1.8 واقتراحات التطوير لـ NetGuard Pro

بعد مراجعة الكود المصدري لمشروع `luci-mobile-1.1.8` ومقارنته بالمشروع الحالي، تم استخراج النقاط والتقنيات التالية التي يمكن أن تدفع بمشروع **NetGuard Pro** إلى مستوى احترافي أعلى:

---

## 1. 🔐 إدارة شهادات SSL (التعامل مع HTTPS)
أكبر عقبة في التعامل مع الراوترات عبر HTTPS هي الشهادات ذاتية التوقيع (Self-signed Certificates).
- **الإضافة المقترحة:** تبني نظام `HttpClientManager` الذي يكتشف فشل المصافحة (Handshake Failure) ويعرض ديالوج للمستخدم لقبول المخاطرة وحفظ الشهادة في `flutter_secure_storage`.
- **الفائدة:** السماح بالاتصال الآمن بالراوتر دون تعطيل الأمان بشكل كامل.

## 2. 📊 دقة قياس البيانات (Throughput Service)
في `luci-mobile` يتم استخدام `Queue<double>` لتخزين سجل البيانات.
- **الإضافة المقترحة:** 
    - استخدام الـ Queue يضمن أداء ثابت O(1).
    - إضافة منطق "Spike Rejection" مبني على السعة القصوى للشبكة (مثلاً 1GB/s كحد أقصى) لمنع القفزات الوهمية الناتجة عن أخطاء الـ Poll latency.
    - دعم مراقبة كل واجهة (Interface) بشكل مستقل (wlan0, eth1, etc) بدلاً من المراقبة الإجمالية فقط.

## 3. 🛡️ اكتشاف البروتوكول التلقائي (Protocol Detection)
المشروع يستخدم وظيفة `loginWithProtocolDetection`.
- **الإضافة المقترحة:** عند محاولة الدخول، يقوم النظام بفحص ما إذا كان الراوتر يرسل `Redirect (302)` من HTTP إلى HTTPS.
- **الفائدة:** توفير تجربة مستخدم سلسة وتلقائية في ضبط الإعدادات الأمنية.

## 4. 🛰️ دعم ميزات LuCI المتقدمة (Extended RPC)
يحتوي المشروع على استدعاءات UBUS جاهزة ومجربة لـ:
- **Wireless Stations:** جلب الماك أدريس للأجهزة المتصلة عبر الواي فاي حصراً باستخدام `iwinfo`.
- **WireGuard:** جلب حالة الـ Peers واستهلاك البيانات الخاص بالـ VPN مباشرة من الراوتر باستخدام `luci.wireguard`.
- **Reboot:** تنفيذ إعادة التشغيل بشكل معياري باستخدام `system/reboot`.

## 5. 🏗️ هيكلة الـ API (Clean Interface)
يستخدم المشروع `IApiService` كواجهة مجردة (Abstract Interface).
- **الإضافة المقترحة:** تسهل هذه الهيكلية إضافة "Mock Mode" (وضع تجريبي) بدون الحاجة لراوتر حقيقي، مما يسهل عمليات التطوير والاختبار.

## 📦 التوصيات النهائية للدمج:
1. **دمج `HttpClientManager`:** لحل مشاكل SSL بشكل نهائي.
2. **تحويل `NetGuardEngine` للعمل بنظام `Interfaces`:** للسماح للمستخدم باختيار أي Interface يريد مراقبتها (مثلاً مراقبة استهلاك الواي فاي فقط).
3. **إضافة ميزة "سجل الأجهزة اللاسلكية":** باستخدام منطق `iwinfo` المستخرج لمنع المتطفلين.
