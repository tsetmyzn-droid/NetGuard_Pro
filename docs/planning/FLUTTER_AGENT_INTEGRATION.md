# 📱 FLUTTER_AGENT_INTEGRATION.md

تصميم طبقة التكامل في تطبيق Flutter للتعامل مع نظام الـ Agent الموزع.

## 🏗️ التصميم الطبقي (Layered Architecture)

### 1. Networking Layer (AgentClient)
- كلاس متخصص (بناءً على Dio) يتعامل مع الـ Agent.
- **المعالجة**: يقوم بإضافة الـ HMAC Signatures تلقائياً لكل طلب عبر interceptor.
- **الأمان**: يتعامل مع التشفير الإضافي إذا لزم الأمر.

### 2. Repository Layer (AgentRepository)
- يوفر وظائف مثل `getHourlyTraffic()`, `getDeviceHistory(mac)`, `getAgentHealth()`.
- **Caching**: يستخدم SQLite محلي (عبر sqflite) لتخزين البيانات التاريخية التي تم جلبها من الراوتر لتسريع العرض وتقليل الطلبات.

### 3. State Management (AgentProvider)
- يدير حالة الاتصال بالوكيل.
- **Stream Support**: يدعم تحديث الواجهة بشكل لحظي عند وصول بيانات جديدة.

---

## 🔄 استراتيجية التزامن (Sync Strategy)

- **Foreground Sync**: عند فتح شاشة "إحصائيات الأجهزة"، يتم جلب آخر ساعة من البيانات.
- **Background Sync**: (اختياري) محاولة جلب الملخص اليومي مرة كل 6 ساعات في الخلفية.
- **Delta Sync**: يتم جلب البيانات الجديدة فقط منذ آخر تحديث (Last Offset) لتقليل حجم البيانات المنقولة.

---

## 📊 رصد البيانات التاريخية (Charts & Analytics)

- استخدام مكتبة `fl_chart` لعرض الرسوم البيانية.
- يتم تحويل الـ DTOs القادمة من الوكيل إلى نماذج بيانات قابلة للرسم بسهولة.
- **Offline Mode**: في حال غياب الراوتر، يمكن للمستخدم تصفح البيانات التاريخية المخزنة سابقاً في التطبيق.

---

## 🛠️ التعامل مع الأخطاء في الواجهة (UI Error Handling)

- **Agent Missing**: عرض زر "تثبيت الوكيل" أو "تفعيل التثبيت السريع".
- **Auth Error**: مطالبة المستخدم بتحديث بيانات الدخول أو إعادة مزامنة المفتاح.
- **Unstable Connection**: إظهار علامة تحذير صفراء مع الاستمرار في عرض آخر بيانات معروفة.

---

## 🧩 التجريد من أجل الإضافات (Abstraction)

واجهة `AgentAdapter` تسمح للتطبيق بالتعامل مع أنواع مختلفة من الوكلاء (OpenWrt Agent, DD-WRT Agent مستقبلاً) بنفس الكود البرمجي في طبقة الـ UI.

---
**نتيجة التصميم**: تكامل عميق وشفاف يوفر للمستخدم شعوراً بأن البيانات موجودة "على جهازه" وليس على الراوتر البعيد.
