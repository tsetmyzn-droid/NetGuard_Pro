# 🚀 AGENT_DEPLOYMENT_STRATEGY.md

استراتيجية نشر وتثبيت NetGuard Agent بشكل آمن وسهل للمستخدم.

## 📦 طرق التثبيت (Installation Methods)

### 1. SSH Automated Install (الخيار المفضل)
- يسمح التطبيق للمستخدم بإدخال بيانات الـ SSH.
- يقوم التطبيق برفع ملف الـ Agent (Lua/Shell) عبر `SCP`.
- ينفذ أوامر الإعداد (Permissions, Web-server config) عبر SSH.
- **الأمان**: لا يتم تخزين كلمة مرور SSH في التطبيق، بل تستخدم للجلسة الحالية فقط.

### 2. Manual LuCI Install (الخيار البديل)
- يوفر التطبيق للمستخدم سكريبت "نسخ ولصق" يتم تنفيذه في واجهة LuCI (Custom Commands).
- يقوم السكريبت بتحميل النسخة المناسبة من الـ Agent من المستودع الرسمي.

---

## 🛡️ النزاهة والتحقق (Integrity Verification)

- **Checksum Matching**: قبل البدء بالتثبيت، يتحقق التطبيق من الـ SHA256 للملف الذي سيتم رفعه.
- **Dry-run Check**: قبل نسخ الملفات، يتم فحص المساحة المتاحة وإصدار الـ Firmware. إذا كان الراوتر غير متوافق (مثلاً إصدار قديم جداً من OpenWrt)، يتم إجهاض التثبيت برسالة واضحة.

---

## ⚙️ خطوات التثبيت الآلي (Pipeline)

1. **Discovery**: اكتشاف نوع الراوتر وإصداره.
2. **Key Exchange**: توليد مفتاح HMAC فريد ورفعه.
3. **Agent Deployment**: نقل سكريبتات Lua و الـ CGI Handler.
4. **Configuration**: تعديل إعدادات `uhttpd` للسماح بمسار `/cgi-bin/netguard`.
5. **Validation**: إجراء أول طلب "Handshake" مع الوكيل الجديد.
6. **Cleanup**: مسح الملفات المؤقتة في `/tmp`.

---

## 🔄 استراتيجية التحديث (Update Strategy)

- **Version Check**: عند كل اتصال، يقارن التطبيق نسخة الوكيل بالنسخة المتاحة لديه.
- **Atomic Update**: يتم تحميل التحديث الجديد في ملف مؤقت. إذا نجح تشغيله، يتم استبدال القديم.
- **Rollback**: في حال فشل التحديث، يتم العودة للنسخة السابقة العاملة وتنبيه المستخدم.

---

## 🩹 آلية الحذف (Uninstallation)

يوفر التطبيق خيار "إزالة الوكيل بالكامل" (Clean Uninstall) الذي يقوم بمسح:
- ملفات الـ Agent.
- ملفات التكوين.
- قواعد بيانات الإحصائيات التاريخية.
- إعدادات uhttpd المضافة.

---
**الضمان**: عملية التثبيت مصممة بحيث لا تتطلب إعادة تشغيل الراوتر (No Reboot Required) ولا تسبب انقطاعاً في الإنترنت حتى لو فشلت العملية في المنتصف.
