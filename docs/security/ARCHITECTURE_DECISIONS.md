# 🏛️ Architecture Decisions (ADR) - NetGuard Pro

## ADR 001: نظام "الوكيل النحيف" (Thin Agent Strategy)
- **القرار**: تقليل المنطق البرمجي داخل الراوتر لأقصى درجة.
- **السبب**: ضمان العمل على الأجهزة الضعيفة (32MB RAM) وتقليل مساحة الهجوم.
- **النتيجة**: معالجة البيانات والتحليلات تتم في جانب التطبيق (Flutter).

## ADR 002: استخدام HMAC للمصادقة المحلية
- **القرار**: عدم الاعتماد على HTTPS فقط داخل الـ LAN.
- **السبب**: تعقيد إدارة شهادات SSL داخل الراوترات الضعيفة.
- **النتيجة**: استخدام HMAC-SHA256 مع Shared Key يتم تبادله أثناء التثبيت عبر SSH.

## ADR 003: التعديلات المرحلية (Transactional Config)
- **القرار**: ممنوع تعديل ملفات الإعدادات مباشرة دون نسخة احتياطية.
- **السبب**: منع فقدان الوصول (Lock-out) عند حدوث خطأ في الإعدادات.
- **النتيجة**: تطبيق الإعدادات لفترة اختبار ثم تأكيدها (Commit) أو التراجع التلقائي.
