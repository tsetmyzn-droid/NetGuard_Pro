================================================================================
                    دليل تطبيق NetGuard Pro - README
              (Guide for Gemini Agent - NetGuard Pro Project)
================================================================================

مرحباً Gemini. هذا الملف هو دليلك الشامل لبناء تطبيق NetGuard Pro.
أرجو قراءته كاملاً قبل أي تعديل. لا تتخطى أي قسم.

================================================================================
  ⚠️ تحذير مهم جداً - اقرأ هذا أولاً ⚠️
================================================================================

1. لا تحذف أي ميزة موجودة في التطبيق. أبداً.
   المستخدم يشتكي أنك تنسى وتحذف الميزات. اقرأ قائمة الميزات أدناه بعناية.

2. مشكلة الشاشة السوداء:
   - المستخدم يقول أن تطبيقك الحالي يعرض شاشة سوداء بها نص فقط عند الضغط يعمل
   - لا توجد أيقونات، ولا ألوان، ولا واجهة رسومية صحيحة
   - هذه المشكلة تحدث عادة بسبب:
     * عدم تحميل Tailwind CSS بشكل صحيح
     * عدم استيراد ملف index.css في index.tsx
     * عدم تطبيق ThemeProvider بشكل صحيح (الثيم dark افتراضي بدون ألوان)
     * استخدام مكونات بدون className أو styles
     * نسيان إضافة Providers للـ Context (I18n, Theme, QueryClient)
   - الحل موجود في قسم "حل مشكلة الشاشة السوداء" أدناه

3. اسم المشروع: NetGuard Pro
   - يجب تغيير كل إشارة من "Router Manager" إلى "NetGuard Pro"
   - لكن احتفظ بالبنية والكود كما هو

4. الهدف النهائي للمستخدم:
   - تطبيق APK (أندرويد) + EXE (ويندوز) يعمل محلياً على جهازه
   - اتصال مباشر بالراوتر على 192.168.1.1 حتى بدون إنترنت
   - تشفير البيانات محلياً
   - التعرف التلقائي على موديل الراوتر
   - العمل على راوترات مصرية (Huawei DG8045, HG630 V2, ZTE ZXHN H168N, إلخ.)
   
   ملاحظة: هذا يتطلب React Native + Electron/Tauri، ليس Modelence.
   إذا كنت تعمل داخل Modelence، فهذا قيد تقني مهم يجب إخبار المستخدم به.

================================================================================
  📋 قائمة الميزات الكاملة - لا تحذف أي منها
================================================================================

الميزات الأساسية (MUST HAVE - لا تحذفها أبداً):

1. دعم اللغتين العربية والإنجليزية (i18n)
   - اتجاه RTL للعربية تلقائياً
   - زر تبديل اللغة في الـ Layout
   - حفظ الاختيار في localStorage

2. الوضع الليلي والنهاري (Dark/Light Mode)
   - زر تبديل الثيم في الـ Layout
   - حفظ الاختيار في localStorage
   - اكتشاف تفضيل النظام تلقائياً
   - CSS class "dark" على element <html>

3. صفحة لوحة التحكم (Dashboard)
   - بطاقات إحصائيات (استهلاك اليوم، الأسبوع، الشهر، عدد الأجهزة)
   - حالة الاتصال بالراوتر
   - الاستهلاك حسب كل جهاز مع progress bars
   - تحديث تلقائي كل 30 ثانية

4. صفحة الأجهزة (Devices Page)
   - شبكة الأجهزة مع أيقونات حسب النوع (هاتف، كمبيوتر، تلفزيون، إلخ)
   - badges للحالة (متصل، محظور، محدود السرعة)
   - قائمة منسدلة لكل جهاز (حظر/فك حظر، تحديد السرعة، إعادة تسمية، حذف)
   - بحث وفلترة

5. صفحة الإعدادات (Settings Page)
   - نموذج الاتصال بالراوتر (IP، اسم المستخدم، كلمة المرور، العلامة التجارية، الموديل)
   - دعم: Huawei, ZTE, TP-Link, D-Link, Netgear, ASUS
   - إعدادات WiFi (SSID، كلمة المرور، القناة، نوع الأمان، إخفاء الشبكة)
   - تحكم في السرعة (Bandwidth)
   - Danger Zone (إعادة تشغيل، ضبط مصنع)

6. صفحة السجلات (Logs Page)
   - قائمة السجلات مع فلتر المستوى (الكل، info، warning، error، success)
   - فلتر الفئة
   - بحث
   - تصدير إلى ملف .txt
   - مسح السجلات القديمة (أكبر من 30 يوم)

7. صفحة الأخطاء (Errors Page)
   - عرض الأخطاء فقط
   - تحديث كل 5 ثوانٍ
   - تجميع حسب التاريخ
   - إحصائيات الفئات
   - عرض الوقت نسبي (time-ago)
   - تبديل الإشعارات
   - مؤشر المراقبة المباشرة

8. نظام المصادقة (Auth)
   - تسجيل الدخول (Login)
   - إنشاء حساب (Signup)
   - تسجيل خروج (Logout)
   - Private routes محمية للمستخدم المسجل

9. قواعد البيانات (Stores):
   - dbRouterConfig: إعدادات الراوتر للمستخدم
   - dbDevices: الأجهزة المتصلة
   - dbUsageStats: إحصائيات الاستهلاك اليومية
   - dbLogs: السجلات
   - dbWifiSettings: إعدادات WiFi
   - dbPackageInfo: معلومات باقة الإنترنت من ISP

10. Queries و Mutations كاملة للعمليات المذكورة أعلاه

الميزات المطلوبة إضافتها (الآن - في NetGuard Pro):

11. مخططات بيانية مفصلة (Detailed Charts)
    - استخدم Recharts أو مشابه
    - line chart للاستهلاك عبر الزمن
    - pie chart لتوزيع الاستهلاك بين الأجهزة
    - bar chart للاستهلاك اليومي/الأسبوعي/الشهري

12. التعرف التلقائي على موديل الراوتر (Auto-Detect Router Model)
    - يحاول الاتصال بـ 192.168.1.1
    - يقرأ صفحة login الخاصة بالراوتر
    - يتعرف على الموديل من HTML/headers
    - يشغل سكريبت مناسب لكل موديل

13. سكريبتات مخصصة لكل موديل (Model-Specific Scripts)
    - Huawei DG8045 (فودافون مصر)
    - Huawei HG630 V2 (اتصالات المنزلية)
    - ZTE ZXHN H168N (WE / المصرية للاتصالات)
    - TP-Link Archer series
    - كل موديل له endpoints وطرق اتصال مختلفة

14. تشفير بيانات المستخدم (User Data Encryption)
    - كلمات مرور الراوتر مشفرة في DB
    - استخدم bcrypt/crypto-js أو similar
    - لا تحفظ أي بيانات حساسة بدون تشفير

15. تحسين الواجهة الرسومية (UI Improvements)
    - إضافة animations smooth
    - إضافة loading states جميلة
    - إضافة empty states مع illustrations
    - إضافة tooltips و hints
    - إضافة keyboard shortcuts

================================================================================
  🔧 حل مشكلة الشاشة السوداء
================================================================================

المستخدم يشكو أن تطبيقك يعرض شاشة سوداء بدون أيقونات وألوان.
هذه قائمة بالأسباب والحلول:

السبب #1: ملف index.css غير مستورد في index.tsx
الحل: 
```typescript
// في /user-app/src/client/index.tsx - في أعلى الملف
import './index.css';  // ← هذا السطر ضروري!
```

السبب #2: Tailwind config لا يكتشف الملفات
الحل:
```javascript
// في /user-app/tailwind.config.js
module.exports = {
  darkMode: 'class',  // ← يجب أن يكون 'class' وليس 'media'
  content: [
    './src/**/*.{js,jsx,ts,tsx}',  // ← جميع الملفات
    './index.html',
  ],
  // باقي الإعدادات...
};
```

السبب #3: Providers مفقودة
الحل في index.tsx:
```typescript
<QueryClientProvider client={queryClient}>
  <ThemeProvider>
    <I18nProvider>
      <RouterProvider router={router} />
    </I18nProvider>
  </ThemeProvider>
</QueryClientProvider>
```

السبب #4: الثيم dark يطبق ألوان سوداء بدون محتوى
الحل: تأكد من وجود bg-background text-foreground على root div

السبب #5: مكونات UI بدون styling
الحل: استخدم الكلاسات من index.css مثل:
- card, btn-primary, btn-secondary, input-field
- badge, badge-success, badge-danger
- sidebar-link, table-header

السبب #6: Routes بدون Layout wrapper
الحل: لف جميع private routes داخل <Layout>:
```typescript
{
  element: <Layout><Outlet /></Layout>,
  children: [...routes]
}
```

================================================================================
  🔐 نظام المصادقة
================================================================================

يستخدم Modelence Auth المدمج:

Client:
```typescript
import { useSession, logout } from 'modelence/client';

const { user } = useSession();  // { id, ... } أو null
await logout();  // تسجيل خروج
```

Server:
```typescript
queries: {
  myQuery: async (args, { user }) => {
    if (!user) throw new AuthError('Not authenticated');
    // user.id متاح
  }
}
```

Routes:
- Public: /terms, /logout
- Guest only: /login, /signup
- Private: /, /devices, /settings, /logs, /errors

================================================================================
  💾 قواعد البيانات (Stores)
================================================================================

كل Store يُعرف في db.ts، ثم يستخدم في index.ts:

```typescript
// في db.ts
import { Store, schema } from 'modelence/server';

export const dbDevices = new Store('devices', {
  schema: {
    userId: schema.userId(),
    macAddress: schema.string(),
    name: schema.string(),
    // ...
  },
  indexes: [{ key: { userId: 1, macAddress: 1 }, unique: true }]
});
```

**مهم جداً**: 
- indexes: `{ key: {...}, unique: true }` وليس `{ options: { unique: true } }`
- لاستعلام عن فيلد optional بقيمة undefined: استخدم `{ $exists: false }` وليس `{ field: null }`

================================================================================
  🌐 Queries و Mutations
================================================================================

Client (استخدام):
```typescript
import { modelenceQuery, modelenceMutation } from '@modelence/react-query';
import { useQuery, useMutation } from '@tanstack/react-query';

// Query
const { data } = useQuery({
  ...modelenceQuery('router.getDevices', {}),
});

// Mutation
const { mutate } = useMutation({
  ...modelenceMutation('router.addDevice'),
});
mutate({ name: 'iPhone' });
```

Server (تعريف):
```typescript
// في index.ts
export default new Module('router', {
  stores: [...],
  queries: {
    getDevices: async (args, { user }) => { ... }
  },
  mutations: {
    addDevice: async (args, { user }) => { ... }
  }
});
```

**مهم**: سجل الـ module في app.ts:
```typescript
import routerModule from './router';
startApp({
  modules: [routerModule, ...]
});
```

================================================================================
  🤖 التعرف التلقائي على موديل الراوتر
================================================================================

**مهم**: هذا لا يعمل من سيرفر Modelence السحابي لأنه لا يستطيع الوصول إلى 
192.168.1.1 الخاص بالمستخدم. يجب تشغيله من المتصفح باستخدام fetch:

```typescript
// في client
async function detectRouter() {
  try {
    const response = await fetch('http://192.168.1.1', { 
      mode: 'no-cors',  // لتجاوز CORS
    });
    const text = await response.text();
    
    if (text.includes('Huawei')) return 'huawei';
    if (text.includes('ZTE')) return 'zte';
    if (text.includes('TP-Link')) return 'tp-link';
    return 'unknown';
  } catch (e) {
    return 'offline';
  }
}
```

**تحذير**: المتصفحات تحجب هذا بسبب CORS. الحل الحقيقي هو React Native / Electron
التي تستطيع عمل HTTP requests بدون قيود.

================================================================================
  🔒 تشفير البيانات (مطلوب)
================================================================================

تثبيت:
```bash
npm install crypto-js
npm install --save-dev @types/crypto-js
```

مثال:
```typescript
import CryptoJS from 'crypto-js';

const SECRET = 'user-specific-key';

export function encrypt(data: string): string {
  return CryptoJS.AES.encrypt(data, SECRET).toString();
}

export function decrypt(encrypted: string): string {
  const bytes = CryptoJS.AES.decrypt(encrypted, SECRET);
  return bytes.toString(CryptoJS.enc.Utf8);
}
```

**يطبق على**: كلمات مرور الراوتر، WiFi passwords، أي بيانات حساسة.
**المفتاح**: يجب أن يكون من getConfig secret في Modelence، ليس hardcoded.

================================================================================
  📱 تحويل إلى APK و EXE (خارج Modelence)
================================================================================

Modelence لا يستطيع بناء APK/EXE مباشرة. الطريقة هي:

1. لـ APK:
   - استخدم Capacitor أو React Native
   - Capacitor: `npm install @capacitor/core @capacitor/cli`
   - `npx cap init NetGuard com.netguard.pro`
   - `npx cap add android`
   - `npx cap sync`
   - فتح المشروع في Android Studio وبناء APK

2. لـ EXE:
   - استخدم Electron أو Tauri
   - Electron: `npm install electron electron-builder`
   - ضع main.js + package.json scripts
   - `npm run build` ينتج EXE

**لكن**: هذه التقنيات خارج بيئة Modelence. يحتاج المستخدم تصدير الكود
وبناء المشروع في بيئة محلية.
